# SPER Agent 集成落地清单（Agentic Engineering 红绿灯法）

## 1. 文档目标

本清单用于将 `D:/project/sper` 的 Agent 能力高效集成到当前 `lumina` 项目，并遵循 Simon Willison 提倡的 Agentic Engineering 实践：  
**把任务按风险分级（红/黄/绿），让 Agent 在可控边界内高效执行，在高风险环节引入强制人工门禁。**

目标不是“接上即可”，而是实现：
- 热点感知增强（选题更准）
- 内容产出增强（改写更优）
- 评估闭环增强（反馈可调权）
- 过程可观测、可回滚、可审计

---

## 2. 红绿灯方法（执行总则）

### 2.1 颜色定义

- **绿灯（Green）**：低风险、可逆、可自动验证。允许 Agent 全自动执行。
- **黄灯（Amber）**：中风险、跨模块、可能影响线上行为。允许 Agent 执行，但必须人工 Review 后再合并/发布。
- **红灯（Red）**：高风险、不可逆或涉及合规/生产数据写入。禁止 Agent 自动执行，必须人工审批和双人复核。

### 2.2 全局门禁

- 所有任务必须绑定：`输入`、`执行动作`、`产出物`、`验收标准`、`回滚策略`。
- 所有上线变更必须支持一键回退（`SPER_ENABLED=false`）。
- 默认 `fail-open`：SPER 失败时自动降级到原 `lumina` 链路，不中断主流程。
- 默认禁止自动调权写入（`apply_changes=false`）。

---

## 3. 集成范围与优先能力

首批接入 4 个高价值 SPER 工具（MVP）：
- `platform_hot_rank_snapshot`（平台热点快照）
- `hotspot_topic_analyze`（热点选题分析）
- `cross_platform_rewrite`（跨平台改写）
- `content_potential_radar`（内容潜力评分）

增强链路（目标形态）：
1. 热点分析  
2. 内容生成/改写  
3. 潜力评分排序  
4. 合规校验  
5. 输出建议与解释

---

## 4. 分阶段可执行清单（按红绿灯标注）

> 标记说明：`[G]` 绿灯，`[Y]` 黄灯，`[R]` 红灯

  - 回滚：无（文档变更）

- [ ] **[G] 任务 0.2：环境基线确认**

  - 产出：`docs/SPER_ENV_CHECKLIST.md`
  - 验收：4 个工具均返回成功或可解释降级结果
  - 回滚：无（环境核验）

## Phase 1：MVP 接入（1 周）

- [ ] **[Y] 任务 1.1：新增 SPER 适配层**
  - 输入：SPER envelope 协议（`success/error_code/data/warnings/request_id`）
  - 动作：在 `apps/api/src/services/handlers/` 增加 `sper_adapter`（统一超时/重试/错误转换）
  - 产出：适配器代码 + 单元测试
  - 验收：
    - 4 个工具本地可调用
    - 统一异常模型可落日志
    - 超时策略生效
  - 回滚：移除路由引用，保留代码不启用

- [ ] **[Y] 任务 1.2：编排接入开关**
  - 输入：`MarketingOrchestra` 现有执行流
  - 动作：增加 `use_sper` 路径（热点 -> 生成 -> 评分）
  - 产出：`apps/orchestra/src/orchestra/core.py` 改造 + 集成测试
  - 验收：
    - 关闭开关时行为与现状一致
    - 开启开关时可返回 SPER 增强结果
  - 回滚：`SPER_ENABLED=false`

- [ ] **[G] 任务 1.3：监控与可观测**
  - 输入：请求上下文（`request_id/user_id/tool_name`）
  - 动作：新增结构化日志和指标埋点
  - 产出：指标面板定义文档 `docs/SPER_OBSERVABILITY.md`
  - 验收：可按工具看成功率、延迟、降级率、错误码
  - 回滚：关闭埋点开关

- [ ] **[Y] 任务 1.4：灰度发布**
  - 输入：测试用户白名单或流量比例
  - 动作：5% -> 20% -> 50% 灰度推进
  - 产出：灰度记录与问题清单
  - 验收：关键 KPI 无明显回退，故障可在 10 分钟定位
  - 回滚：流量切回 0%

## Phase 2：效果闭环（2-3 周）

- [ ] **[Y] 任务 2.1：评分驱动候选排序**
  - 输入：多候选生成结果
  - 动作：调用 `content_potential_radar` 进行 TopN 排序
  - 产出：在 `skill-bulk-creative`、`skill-creative-studio` 的排序逻辑
  - 验收：输出附带评分解释与推荐理由
  - 回滚：退回原排序策略

- [ ] **[R] 任务 2.2：残差反馈调权**
  - 输入：发布后真实表现数据（点击/完播/互动）
  - 动作：接入 `residual_feedback_update`，先仅生成建议不自动写入
  - 产出：周度调权建议报告
  - 验收：至少 2 周观察到可解释改善趋势
  - 回滚：关闭反馈入口，恢复基线权重
  - 强制门禁：人工审批 + 双人复核后才允许写入

- [ ] **[Y] 任务 2.3：合规与评分联合裁决**
  - 输入：平台规则 + 潜力评分
  - 动作：增加“先合规后评分”判定器
  - 产出：联合判定逻辑与错误提示文案
  - 验收：不存在“高分但违规仍放行”的结果
  - 回滚：回退到现有合规链路

  - 产出：`docs/SPER_AB_REPORT.md`

- [ ] **[Y] 任务 3.3：数据契约治理**

  - 回滚：兼容层回退到旧版本 schema

## 5. 配置与开关（上线最小控制面）

建议新增以下配置：
- `SPER_ENABLED`：是否启用 SPER 增强链路（默认 false）
- `SPER_TIMEOUT_MS`：SPER 调用超时
- `SPER_RETRY_COUNT`：重试次数
- `SPER_FAIL_OPEN`：失败是否自动回退到原链路（默认 true）
- `SPER_WRITEBACK_ENABLED`：是否允许调权写入（默认 false）

上线要求：
- 任一配置切换均可热更新或快速重启生效
- 所有配置修改有审计日志

## 7. 运行手册（值班视角）

故障优先级处理：
1. 先看 `SPER_FAIL_OPEN` 是否生效（确保业务不中断）
2. 再看工具级成功率和错误码分布
3. 识别是单工具故障还是全链路故障
4. 必要时关闭 `SPER_ENABLED` 完成全局回退

值班触发条件（建议）：
- 单工具 5 分钟成功率 < 95%
- 全链路降级率 > 20%
- P95 延迟持续 15 分钟超阈值

---

## 8. 成功标准（Definition of Done）

满足以下条件即可判定“集成完成且高效利用新 Agent 能力”：
- 4 个 MVP 工具稳定服务，成功率 >= 98%
- 关键链路具备完整可观测与快速回退能力
- A/B 实验显示业务指标显著正向（至少 2 周）
- 调权能力在红灯门禁下可控运行
- 团队有可执行文档、测试与值班手册，不依赖单点人员

---

## 9. 执行建议（团队协作）

- 每周固定节奏：周一定范围、周三看指标、周五复盘与调参
- 所有红灯任务必须创建审批记录与回滚演练记录
- 对 Agent 的要求是“高自动化 + 强约束”，不是“全自动无门禁”

> 结论：采用红绿灯法后，`sper` 集成可以在保持风险可控的前提下，最大化放大 `lumina` 的热点、创作和评估闭环能力。
